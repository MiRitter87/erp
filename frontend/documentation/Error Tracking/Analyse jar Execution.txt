Problem:
WebService aus den generierten Klassen kann im Client nicht initialisiert werden, wenn man die fertige .jar per Doppelklick öffnet.

-Die Umstellung von einer lokalen WSDL auf eine Remote WSDL brachte keine Besserung
-Die Ausführung der jar über die Konsole mit Verbindung eines Remote Debuggers lässt das Problem nicht erscheinen
-Die Ausführung der jar per Konsole über java -jar lässt das Problem nicht erscheinen

-Problem erscheint nur bei Ausführen per Doppelklick auf .jar Datei

-Idee: Andere Ausführungsumgebung zur Ausführung nutzen. Nämlich die, die auch über die Konsole genutzt wird
-java -version gibt folgende Info zur Ausführungsumgebung:
java version "1.8.0_261"
Java(TM) SE Runtime Environment (build 1.8.0_261-b12)
Java HotSpot(TM) 64-Bit Server VM (build 25.261-b12, mixed mode)
-Bei Doppelklick wird genutzt: OpenJDK Platform Binary
